451. N-Queens

🔗Question Link

DFS。queen游戏规则是同一列/同一行/正对角/斜对角不能有Q存在，由于我们根据行数来遍历dfs，所以检查valid函数要考虑（列/正对角/斜对角）三种情况。

52. N-Queens II

🔗Question Link

DFS。和I不同之处是要返回可能的情况个数，但是不用像I一样麻烦。思路一样，对三个方向检查，true表明有Q，false表明没有Q，创建三个方向的boolean数组，这里要注意正/斜对角的个数以及index和row，col的关系，画个图就能明白。

37. Sudoku Solver

🔗Question Link

DFS。类似queen的题。遍历图，对‘.‘的点进行dfs。从1-9进行选择，然后检查当前选择的数是否可以放，如果可以则调用solve函数继续dfs。如果不行则把当前位置变回’.’。

难点在valid函数里对每一个3x3方格的检查。行用/ 列用%。

22. Generate Parentheses

🔗Question Link

dfs。left和right都为n个，则把list加入res。只有当right<left时才递归加入右括号。

542. 01 Matrix

🔗Question Link

bfs。先把0的坐标加进queue，把不为零的位置值变为-1。从外围往里扫描，如果更新后的位置不为-1，说明没有找到不为零的位置，继续扫描。如果更新后的位置为-1，说明当前位置是0，所以把更新后位置的值变为当前位置+1，依次类推。

934. Shortest Bridge

🔗Question Link

dfs + bfs。 dfs来找任意一个1所属的island的所有1的坐标。bfs则用来根据当前找到的island所有坐标，来找能连接到另外一个岛的1所需要的steps。dfs过程中需定义一个boolean值来判断是否找到全部的岛，如果true则直接进入bfs。
